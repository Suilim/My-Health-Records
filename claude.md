# 我的健康紀錄 App

## 專案目標
- 幫助精神疾病使用者簡單記錄每日健康狀態。
- 建立使用者個人健康資料庫。
- 資料庫可匯出的格式交給機構。
- 串接語言模型進行近3個月資料摘要，

## 技術棧
- Python + Streamlit
- 資料存取：Firebase，
- 部署方式：構思中
- 使用uv管理環境

## 檔案結構
- pages：各項紀錄頁面
- pages/0_⚙️_設定.py：讓使用者選擇要紀錄的模組、是否設定補填提醒、提醒開啟天數、用藥時段組合設定等
- pages/1_💊_用藥.py：用藥紀錄填寫、自動帶入之前同時段用藥紀錄免去重複登打
- pages/2_❤️_血壓.py：血壓紀錄填寫
- pages/3_⚖️_體重.py：體重紀錄填寫、可以選填是否填寫BMI、體脂等數據
- pages/4_🩸_血糖.py：血糖紀錄填寫
- pages/5_🌡️_體溫.py：體溫紀錄填寫
- pages/6_🏃_生活.py：心情與日記填寫
- pages/7_🤧_症狀.py：症狀紀錄填寫，支援多筆新增、發生時間、持續時間選擇題、今日無症狀快速打勾、套用上次清單
- pages/8_📊_圖表與匯出.py：健康數據儀表板與匯出成excel的功能
- pages/9_📅_歷史總覽.py：（是一個測試功能頁面）讓紀錄以日曆方式呈現
- app.py：首頁，先登入，再依據設定模組與提醒設置設定補填通知按鈕、顯示今日的填寫按鈕

## 重要設計原則
- 使用者為精神疾病患者，UI 必須簡單清楚，有streamlit-ui-designer可供調用。
- 使用者多以手機為主，請考慮在手機畫面的呈現

## 工作守則
- 每次完成功能後，主動更新本文件的開發進度
- 如有新建立核心工具模組，請更新本文件的核心工具模組區塊
- 如有覺得本文件有記載不清楚的地方，可適度補充說明

## 核心工具模組
- `firebase_utils.py`：Firebase 資料庫連線與初始化
- `write_records.py`：各項健康數據的寫入邏輯
- `check_records.py`：紀錄讀取與狀態檢查
- `settings_utils.py`：使用者設定、模組啟用、提醒相關函數
- `export_records.py`：匯出紀錄為 Excel
- `plot_utils.py`：圖表與儀表板的繪圖函數（供 7_📊_圖表與匯出.py 使用）
- `calendar_utils.py`：日曆總覽相關函數（供 7_📅_歷史總覽.py 使用）
